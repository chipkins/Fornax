Vulkan SDK for Linux Release Notes

Vulkan SDK 1.0.68.0 Release Notes

Version 1.0.68.0 for Linux
February 8, 2018

This SDK supports Vulkan API revision 1.0.68. The prior SDK supported Vulkan API revision 1.0.65.

Dependencies for Ubuntu now include:

libx11-dev libxcb1-dev libxkbcommon-dev libmirclient-dev libwayland-dev

The dependencies on other platforms may be different.

Please refer to the Getting Started Guide for a list of supported Linux distributions.

Drivers: LunarG recommends that you update your Vulkan-capable hardware drivers to the latest recommended version from your hardware vendor. A list of Known Driver Issues appears at the end of this document with more information.

Linux Loader now searches /usr/local/etc/vulkan and /usr/local/share/vulkan for drivers and layers. These new paths are in addition to the previous locations of /etc/vulkan, /usr/share/vulkan, and the user's HOME local path, e.g., $HOME/.local/share/vulkan. The two new "/usr/local" locations are configurable at build time, allowing configuration of the loader for specific filesystem structures. See the BUILD.html guide in the Loader And Validation Layers repository and the "Loader and Layers Interface" guide for more information.

CMake: CMake 3.2 or later is now required to build the source code contents of the SDK. Any user with an older version will need to upgrade.

Overview of new features in SDK 1.0.68

  • Improved SDK documentation structure
  • The VulkanInfo utility now features an html backend, improving information accessibility
  • The Vulkan Assistant Layer
  • The Vulkan Layer Framework
  • The Vulkan Loader now support pre-Instance intercepts by layers
  • Support for the following extensions:
    • VK_EXT_conservative_rasterization
    • VK_EXT_external_memory_dma_buf
    • VK_EXT_queue_family_foreign
    • VK_EXT_external_memory_host
  • Scores of bug fixes, validation coverage and accuracy improvements, and many feature additions

Developer Notes

  • The pre-compiled glslang static libraries are not compatible with the Linux 4 runtime (e.g., Ubuntu 15.10). These libraries must be rebuilt before use with the SDK samples. See the Vulkan Samples page for more information.
  • The script source/shaderc/update_shaderc_sources.py fails on Arch Linux because python3 is the default python interpreter. To work around this issue, change the line at the top of the file to be python2.

Important Notes About Experimental Extensions

The KHX and NVX identifiers in the name indicate that the extension is experimental, and is being considered for future standardization. In the case of a KHX extension, this extension is being considered as either a KHR extension or core Vulkan API functionality. NVX extensions are being considered for either a final NV or a more general (EXT, KHR) extension.

As experimental extensions:

  • Developers are encouraged to experiment with them and provide feedback, but should not use them as the basis for production applications.
  • These experimental extensions are expected to be supported for a limited time only.
  • They may change their interfaces and behavior in significant ways as a result of feedback
  • They may be withdrawn or replaced with a different stable (KHR, EXT, ...) extension or core functionality at any time.
  • Implementations of these extensions receive limited or no testing when submitted to the Khronos conformance process.
  • Using experimental extensions with the validation layers may produce inaccurate validation results.
